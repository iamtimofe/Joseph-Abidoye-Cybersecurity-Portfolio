# Final Incident Report

## Incident Summary

This investigation was conducted to evaluate the effectiveness of Microsoft Sysmon in monitoring endpoint activity on a Windows Server 2022 system.

Sysmon was successfully deployed using the SwiftOnSecurity configuration, and multiple security-relevant events were generated to simulate common endpoint activity. The resulting telemetry was collected and analyzed through Event Viewer to demonstrate how Security Operations Center (SOC) analysts investigate endpoint behavior.

---

# Investigation Objectives

- Install Microsoft Sysmon
- Configure endpoint monitoring
- Generate endpoint telemetry
- Analyze Sysmon events
- Identify Indicators of Compromise (IOCs)
- Correlate endpoint activity
- Document findings

# Investigation Findings

## Event ID 1 – Process Creation

Multiple legitimate Windows processes were executed during testing. Sysmon successfully recorded the executable path, parent process, command-line arguments, process identifiers, and user context.

---

## Event ID 3 – Network Connection

Outbound network activity generated by Windows applications was successfully captured. Sysmon identified the originating process and recorded destination IP addresses and ports.

---

## Event ID 11 – File Creation

Sysmon detected file creation activity during the download of the Sysmon configuration file.

The event recorded:

- Process responsible for creating the file
- Target filename
- Process GUID
- Timestamp

The generated Zone.Identifier Alternate Data Stream confirmed that Windows recognized the file as originating from the Internet.

---

## Event ID 22 – DNS Query

DNS requests generated during testing were successfully captured.

The event included:

- Process initiating the query
- Domain queried
- Query status
- DNS response

This telemetry demonstrates how Sysmon can assist in identifying suspicious domain lookups.

# Lessons Learned

This investigation demonstrated the value of Sysmon in providing enhanced endpoint telemetry for threat hunting and incident response.

The collected events enabled visibility into:

- Process execution
- Network communications
- File creation activity
- DNS resolution

These data sources are essential for detecting malicious behavior and reconstructing attacker activity during an investigation.

---

# Recommendations

- Forward Sysmon logs to a SIEM such as Wazuh or Splunk.
- Enable centralized log collection and alerting.
- Develop Sigma detection rules for common attack techniques.
- Integrate threat intelligence feeds for IOC enrichment.
- Regularly review Sysmon configuration to ensure comprehensive endpoint visibility.

---

# Conclusion

The Sysmon Endpoint Monitoring Lab successfully demonstrated how enhanced Windows logging improves endpoint visibility and supports threat detection.

The investigation produced actionable telemetry across process creation, network communication, file creation, and DNS activity, providing a realistic foundation for SOC monitoring and threat hunting workflows.
