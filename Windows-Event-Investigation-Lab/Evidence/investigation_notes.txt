Windows Event Investigation Lab

Phase 1 – Failed Logon Investigation

Event ID:
4625

Event Summary:
A failed logon event was generated by intentionally entering an incorrect password for the local Administrator account.

Key Findings:
- Target Account: Administrator
- Failure Reason: Unknown user name or bad password
- Status: 0xC000006D
- Sub Status: 0xC000006A
- Logon Type: 7 (Workstation Unlock)
- Source IP: 127.0.0.1
- Authentication Package: Negotiate
- Process: svchost.exe

SOC Analysis:
The failed logon originated from the local machine (127.0.0.1) and was intentionally generated as part of this lab. No indicators suggest malicious activity. In a production environment, repeated Event ID 4625 entries from the same source or targeting privileged accounts could indicate brute-force or password-spraying attacks and should be investigated further.

Phase 2 – Successful Logon Investigation

Event ID:
4624

Event Summary:
A successful logon event was identified in the Windows Security log.

Observed Event:
- Logon Type: 5 (Service Logon)
- Account Name: SYSTEM
- Account Domain: NT AUTHORITY
- Authentication Package: Negotiate
- Process: C:\Windows\System32\services.exe

SOC Analysis:
This event represents a Windows service authentication rather than an interactive user login. It is considered normal operating system activity and does not indicate suspicious behavior.

Phase 3 – User Account Creation Investigation

Event ID:
4720

Event Summary:
A new local user account named SOC_Analyst was created on the Windows Server.

Key Findings:
- Creating Account: Administrator
- New Account: SOC_Analyst
- Account Domain: WIN-OBNND2CCKKS
- User Account Type: Normal Account
- Account Status: Disabled
- Password Configuration: Password Not Required

SOC Analysis:
The account creation was performed intentionally during this lab by the local Administrator account. In a production environment, unexpected account creation events should be verified against approved administrative changes. Unauthorized account creation may indicate persistence or privilege escalation by an attacker.

Phase 4 – Privilege Escalation Investigation

Event ID:
4732

Event Summary:
The local user account SOC_Analyst was added to the built-in Administrators group.

Key Findings:
- Action Performed By: Administrator
- Account Added: SOC_Analyst
- Group: Administrators
- Group Type: Security-enabled local group

MITRE ATT&CK:
- T1098 – Account Manipulation
- T1078 – Valid Accounts

SOC Analysis:
The account was intentionally added to the Administrators group during the lab. In a production environment, unexpected additions to privileged groups should be treated as high-priority security events and verified immediately.

Recommendation:
Monitor privileged group membership changes, review the initiating account, and validate that the change was authorized.
